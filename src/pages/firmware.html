y_firm_darkfox Architectural Matrix

This document details the low-level bare-metal execution parameters for the 3DS dual-core environment, bypassing the standard operating system layers.

Processor Execution & Memory Map

  • ARM9 Core (ARMv5TE): Supervisor core executing code directly at entry vector 0x08000000 (SysIRAM). Responsible for display initialization, D-Pad register parsing via REG_PAD_HID, and direct VRAM rendering.
  • ARM11 Core (ARMv6K): Secondary application core parked at execution vector 0x1FF80000 (AXI IRAM) inside a low-power wfi (Wait For Interrupt) assembly loop.
  • Graphics Subsystem: Linear 32-bit frame buffer memory mapped directly to the top screen VRAM address space at 0x18000000.

FIRM Layout & Cryptographic Verification

  • Sector Constraint: Mandates strict 512-byte (0x200) block padding across all compiled payloads to comply with hardware NAND/SD interface controllers.
  • Integrity Tags: Standardized SHA256 checksums are calculated over individual blocks and hardcoded into the FIRM metadata layout.
  • Crypto Bypass: Allocates a dedicated 256-byte (0x100) dummy RSA signature field initialized to null blocks to satisfy Luma3DS bootloader structures without triggering format corruption faults.
